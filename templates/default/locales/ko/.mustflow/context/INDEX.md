# 문맥 색인

이 파일은 현재 작업에 필요한 프로젝트 문맥 파일을 결정하고 참조하기 위한 안내서입니다.
불필요한 정보 유입을 방지하기 위해 기본적으로 모든 문맥 파일을 읽지 않습니다.

## 사용 가능한 문맥

| 문맥 | 적용 시나리오 | 경로 |
| --- | --- | --- |
| project | 작업이 프로젝트 방향, 범위, 공개 동작, 비목표, 저장소 전반의 규칙에 잠재적으로 영향을 미치는 경우 | `.mustflow/context/PROJECT.md` |

## 선택적 외부 참조

| 앵커 | 적용 시나리오 | 경로 |
| --- | --- | --- |
| 사용자용 개요 | 공개 프로젝트 개요나 설치 안내가 필요한 경우입니다. 이를 강제적인 정책이 아닌 일반적인 문맥으로 참조하십시오. | `README.md` |
| 시각 디자인 | UI, 시각적 정체성, 디자인 토큰, 레이아웃 또는 접근성을 변경하는 경우 | `DESIGN.md` |

## 읽기 규칙

- 현재 작업에 해당하는 문맥 파일만 참조하십시오.
- 문맥 파일은 더 높은 권한의 출처에 기반한다고 명시적으로 기재되지 않은 경우, 참고용 지침으로 간주합니다.
- 문맥이 코드, 테스트, 명령 계약, 명시적 사용자 지시와 충돌하는 경우, 이를 보고하고 더 높은 권한의 출처를 우선하십시오.
- 누락된 프로젝트 목표, 비목표, 디자인 토큰, API 계약, 데이터 규칙을 임의로 추측하거나 생성하지 마십시오.
- `DESIGN.md`의 디자인 토큰을 `.mustflow/context/` 내에 중복하여 기록하지 마십시오.
