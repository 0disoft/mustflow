# mustflow Examples

This directory contains human-readable project examples.

Examples are not test fixtures and are not copied by `mf init`. They show the shape
of a project before and after mustflow is introduced, while keeping generated or
template-owned files out of the example body.

Current examples:

- `docs-only/`: A documentation-focused repository with explicit missing code checks.
- `host-instruction-conflicts/`: A repository where host-specific instructions and mustflow rules overlap.
- `minimal-js/`: A small JavaScript package before and after adding mustflow.
- `missing-command-contracts/`: A repository with package scripts that are not yet mustflow command intents.
- `nested-repos/`: Parent and child repositories with separate local contracts.
