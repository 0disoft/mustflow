`.mustflow/skills/INDEX.md`는 에이전트가 반복 작업에 들어가기 전에 어떤 스킬 문서를 읽어야 하는지 판단하게 돕는 색인입니다.

## 어디에 쓰이나

에이전트는 공통 규칙과 명령 계약을 읽은 뒤, 현재 작업이 미리 정의된 절차에 해당하면 이 색인에서 맞는 스킬을 찾습니다.

이 파일은 스킬 본문을 길게 복사하지 않습니다. 대신 트리거, 필요한 입력, 수정 범위, 위험, 검증 의도, 예상 출력이라는 간결한 라우팅 항목을 스킬 경로에 연결합니다.
`mf check --strict`는 이 연결 정보를 실제 `SKILL.md`와 비교해 없는 스킬 문서, 색인에 빠진 스킬, 알 수 없는 명령 의도, 스킬 본문 메타데이터와 맞지 않는 명령 의도, 라우팅 표 구조의 어긋남을 드러냅니다.

## 선택 방식

에이전트는 작업 시작 시점과 첫 수정 전에 이 색인을 사용합니다. 사용자 요청과 예상 변경
파일을 나열된 트리거와 비교하고, 맞는 `SKILL.md`를 읽은 뒤 해당 범위를 수정합니다.

명령 실패, 테스트 계약 변경, 문서 변경처럼 작업 중 새 조건이 생기면 잠시 멈추고
새로 맞는 스킬을 읽은 뒤 계속합니다.

맞는 트리거가 없으면 스킬을 추측해서 만들거나 적용하지 않습니다. `AGENTS.md`,
`.mustflow/docs/agent-workflow.md`, `.mustflow/config/commands.toml`을 기준으로 진행합니다.

## 역할

- 스킬 목록과 적용 트리거를 한곳에 모읍니다.
- 각 경로에 필요한 입력, 수정 범위, 위험, 예상 출력을 붙여 범위를 좁힙니다.
- 각 스킬에서 참조할 명령 의도 이름을 함께 보여줍니다.
- 절차 세부 내용은 각 `SKILL.md`에 두고, 색인은 빠르게 탐색할 수 있게 유지합니다.

## 작성 기준

색인은 짧고 빠르게 탐색할 수 있어야 합니다.

각 스킬의 자세한 절차는 해당 `SKILL.md`에 두고, `INDEX.md`에는 에이전트가 어떤 스킬을 읽을지와 어떤 근거를 보고할지 판단하는 데 필요한 라우팅 항목만 둡니다.

## 표 항목

- `트리거`: 에이전트가 스킬을 읽어야 하는 작업 조건입니다.
- `스킬 문서`: 실제 절차가 들어 있는 `SKILL.md` 경로입니다.
- `필요한 입력`: 스킬 적용 전에 필요한 요청 정보나 근거입니다.
- `수정 범위`: 스킬이 안내할 수 있는 파일이나 표면입니다.
- `위험`: 이 경로가 줄이려는 주요 실패 유형입니다.
- `검증 의도`: 해당 스킬에서 관련될 수 있는 `commands.toml`의 의도 이름입니다.
- `예상 출력`: 스킬을 사용한 뒤 에이전트가 남겨야 하는 보고 형태입니다.

새 스킬을 추가하면 이 색인에 경로를 추가하고, 검증 의도 이름이 실제 스킬 앞부분 메타데이터와 맞는지 확인합니다.
