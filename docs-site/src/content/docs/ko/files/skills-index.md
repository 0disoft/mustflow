`.mustflow/skills/INDEX.md`는 에이전트가 반복 작업에 들어가기 전에 어떤 스킬 문서를 읽어야 하는지 판단하게 돕는 색인입니다.

## 어디에 쓰이나

에이전트는 공통 규칙과 명령 계약을 읽은 뒤, 현재 작업이 반복 절차에 해당하면 이 색인에서 맞는 스킬을 찾습니다.

이 파일은 스킬 본문을 길게 복사하지 않고, 상황·스킬 문서 경로·관련 명령 의도만 연결합니다.

## 선택 방식

에이전트는 작업 시작 시점과 첫 수정 전에 이 색인을 사용합니다. 사용자 요청과 예상 변경
파일을 아래 시나리오와 비교하고, 맞는 `SKILL.md`를 읽은 뒤 해당 범위를 수정합니다.

명령 실패, 테스트 계약 변경, 문서 변경처럼 작업 중 새 조건이 생기면 잠시 멈추고
새로 맞는 스킬을 읽은 뒤 계속합니다.

맞는 시나리오가 없으면 스킬을 추측해서 만들거나 적용하지 않습니다. `AGENTS.md`,
`.mustflow/docs/agent-workflow.md`, `.mustflow/config/commands.toml`을 기준으로 진행합니다.

## 역할

- 스킬 이름과 사용 조건을 한곳에 모읍니다.
- 코드 검토, 문서 수정, 실패 추적, 테스트 유지보수처럼 자주 반복되는 작업을 연결합니다.
- 각 스킬에서 참조할 명령 의도 이름을 함께 보여줍니다.
- 저장소에서 쓰지 않는 스킬은 제거하거나 비활성 상태로 표시할 수 있습니다.

## 작성 기준

색인은 짧고 빠르게 탐색할 수 있어야 합니다.

각 스킬의 자세한 절차는 해당 `SKILL.md`에 두고, `INDEX.md`에는 이름, 목적, 읽어야 하는 조건, 관련 명령 의도만 둡니다.

## 표 항목

- `상황`: 에이전트가 스킬을 읽어야 하는 작업 조건입니다.
- `읽을 문서`: 실제 절차가 들어 있는 `SKILL.md` 경로입니다.
- `관련 명령 의도`: 해당 스킬이 확인할 수 있는 `commands.toml`의 의도 이름입니다.

새 스킬을 추가하면 이 색인에 연결하고, `command_intents`는 실제 스킬 앞부분 메타데이터와 맞춥니다.
