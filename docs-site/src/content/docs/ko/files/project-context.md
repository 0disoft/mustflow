`.mustflow/context/PROJECT.md`는 `mf init`이 설치하는 기본 프로젝트 문맥 파일입니다.

이 파일은 짧게 유지해야 합니다. 전체 아키텍처 문서, 로드맵, API 목록, 회의록, 자동 생성 요약 보관함이 아닙니다.

## 어디에 쓰이나

- 작업이 범위, 동작, 저장소 전체 약속에 영향을 줄 수 있을 때 에이전트에게 프로젝트 방향을 알려줍니다.
- 비목표를 기록해 에이전트가 관련 없는 작업을 넓히지 않게 합니다.
- 구현 판단에 영향을 주는 도메인 용어와 특히 조심할 영역을 적습니다.

## 권위

기본 권위는 `contextual`입니다.

즉 이 파일은 에이전트의 방향 파악을 돕지만, 사용자 직접 지시, 현재 코드, 테스트, 명령 계약, 설정된 정책보다 낮은 권위를 가집니다.

현재 파일과 충돌하면 에이전트는 충돌을 보고하고 이 문맥을 오래된 내용으로 봐야 합니다.

## 구성

- `현재 목표`: 현재 프로젝트 목표입니다. 모르면 추측하지 말고 미설정으로 둡니다.
- `비목표`: 관련 없는 작업 중 확장하지 말아야 할 범위입니다.
- `핵심 약속`: 에이전트가 지켜야 하는 저장소 전체 약속입니다.
- `도메인 용어`: 구현 판단에 영향을 주는 용어입니다.
- `특히 조심할 영역`: 주의해야 할 경로, 공개 API, 생성 파일, 마이그레이션, 비밀정보, 호환성 표면입니다.
- `다음으로 읽을 파일`: 이 문맥 뒤에 확인할 파일입니다.
- `오래된 내용 확인`: 이 파일이 낡았는지 판단하는 기준입니다.
