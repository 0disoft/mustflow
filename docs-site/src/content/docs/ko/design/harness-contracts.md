---
title: 하네스 계약
description: mustflow가 장기 실행 에이전트 하네스를 직접 만들지 않고 지원하는 방식입니다.
---

mustflow는 자율 장기 실행 에이전트를 직접 호스팅하지 않습니다. 대신 그런 에이전트
하네스가 저장소에서 읽고 검증할 수 있는 저장소 내부 계약을 제공합니다.

## 경계

- mustflow는 작업자, 페르소나, 다중 에이전트 조정, 클라우드 샌드박스를 직접 제공하지 않습니다.
- mustflow는 원시 세션 로그를 무한히 저장하지 않습니다.
- mustflow는 호스팅 에이전트 플랫폼이나 편집기 에이전트를 대체하지 않습니다.
- mustflow는 규칙, 명령 계약, 지침 새로고침 지점, 압축 기억 정책, 실행 영수증, 예산, 승인, 인계 경계를 정의합니다.

## 사고, 실행, 세션

- 사고 계약: `AGENTS.md`, `agent-workflow.md`, `skills/*/SKILL.md`입니다.
- 실행 계약: `commands.toml`, 단발성 명령 생명주기, `mf run`입니다.
- 세션 계약: 제한된 실행 영수증, 선택형 체크포인트, 출처가 있는 압축 요약, 짧은 인계 문서, 재생성 가능한 색인입니다.
- 판정 계약: 원래 완료 기준, 변경 파일, 명령 계약, 실행 영수증입니다.

이 접근은 mustflow를 특정 도구에 묶지 않습니다. 호스트가 단일 채팅 세션이든, 백그라운드
클라우드 에이전트든, 외부 반복 실행기든 저장소 계약은 같은 방식으로 읽힙니다.

## 지금 채택한 것

- `.mustflow/config/mustflow.toml`의 `[harness]`, `[budget]`, `[approval]`, `[isolation]`
  정책 필드
- `agent-workflow.md`의 검증 약화 방지 규칙
- 지침 새로고침 지점, 계층형 압축 기억 정책, 제한된 보존 정책

압축 요약은 현재 사용자 지시, 현재 코드와 설정, 명령 계약, 실행 영수증보다 낮은 보조
자료입니다. mustflow는 숨겨진 내부 추론이나 대화 원문 전체를 프로젝트에 저장하지 않습니다.

## 뒤로 미룬 것

`completion-judge`, 작업 항목, 인계 작성 명령, 체크포인트 명령, 자율 반복 실행기는
선택형 후보로 둡니다. 이 기능들이 의존하는 계약이 안정되기 전에는 기본 템플릿에 넣지
않습니다.
