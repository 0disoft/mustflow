---
title: mf dashboard
description: 로컬 mustflow 대시보드를 시작합니다.
---

`mf dashboard`는 mustflow 상태, 검증 추천, 명령 의도 확인, 릴리스와 버전 소스 상태, 템플릿 업데이트 가능 여부, 최근 실행 기록, 스킬 라우트, 안전한 선호 설정, 문서 검수 목록을 다루는 로컬 대시보드 서버를 시작합니다.

상태 탭은 설치 여부, 잠금 파일, 템플릿, 변경되거나 누락된 추적 파일, 실행 가능한 명령 수, 최근 실행 기록, 진행 중인 문서 검수 수를 보여줍니다. 검증 추천 탭은 변경 파일을 읽고 복사할 수 있는 `mf run ...` 명령 의도를 추천하되 직접 실행하지는 않습니다. 명령 탭은 `.mustflow/config/commands.toml`을 읽어 실행 가능한 명령, 사용자 요청이 필요한 명령, 아직 설정되지 않은 명령, 차단된 명령을 보여주되 직접 실행하지는 않습니다. 릴리스 탭은 설치된 mustflow 버전, 템플릿 버전, 감지된 버전 소스, 릴리스 관련 변경 파일, 복사할 수 있는 릴리스 검증 명령을 보여줍니다. 업데이트 탭은 `mf update --dry-run` 계획, 차단 항목, 적용될 템플릿 변경을 보여주되 직접 적용하지는 않습니다. 실행 기록 탭은 `.mustflow/state/runs/latest.json`을 읽어 최근 `mf run` 기록의 상태, 시간, 명령줄, 종료 정보, 보존된 출력 일부를 보여줍니다. 스킬 탭은 `.mustflow/skills/INDEX.md`를 읽어 각 라우트의 트리거, 범위, 위험, 명령 의도 일치 여부를 보여줍니다. 설정 탭은 `.mustflow/config/preferences.toml`을 수정합니다. 문서 검수 탭은 `.mustflow/review/docs.toml`을 읽고, 검수 코멘트가 있으면 함께 보여주며, 기존 검수 항목을 승인, 무시, 사람 검수 필요 상태로 표시할 수 있습니다. 스테이징, 커밋, 푸시, 버전 올림, 템플릿 업데이트 적용, 명령 의도 실행은 하지 않습니다.

수정 가능한 그룹은 Git 기본값, 커밋 메시지 제안, 보고, 검증 선택, 테스트 작성, 리팩토링 후보 탐지 기준과 제한값, 코드 스타일, 버전 영향 선호값입니다.

## 현재 동작

```sh
npx mf dashboard
```

이 명령은 기본적으로 `127.0.0.1`에 로컬 HTTP 서버를 띄우고 대시보드 주소를 출력합니다. `--open`을 지정하지 않으면 브라우저를 열지 않습니다.

대시보드 화면에서는 영어, 한국어, 중국어, 스페인어, 프랑스어, 힌디어 중 표시 언어를 바꿀 수 있습니다. 선택한 언어는 브라우저에 저장됩니다.

상태 탭은 읽기 전용입니다. `mf status`, `mf context`, 문서 검수 대기열과 같은 로컬 상태를 사용하므로 설정을 바꾸기 전에 현재 작업 흐름의 모양을 먼저 확인할 수 있습니다.

검증 추천 탭은 읽기 전용입니다. 변경 파일을 mustflow 워크플로, 문서, 코드, 릴리스 민감 영역으로 분류한 뒤, 어떤 설정된 명령 의도가 왜 추천됐는지 보여줍니다.

명령 탭도 읽기 전용입니다. 각 명령 의도의 상태, 수명 주기, 실행 정책, 표준 입력 설정, 제한 시간, 실행 위치, 쓰기 경로, 선언된 차단 이유를 보여줍니다. `manual_only` 명령은 고장 나거나 없는 명령처럼 보이지 않도록 `사용자 요청 필요`로 표시합니다.

릴리스 탭도 읽기 전용입니다. 로컬 버전 소스 감지와 명령 계약 메타데이터를 사용해 릴리스 관련 작업을 받아들이기 전에 무엇을 확인해야 하는지 보여줍니다. `mf version --check`, `mf run test_release`, `mf run docs_validate` 같은 명령은 복사할 수 있지만, npm에 자동으로 접속하거나 명령을 직접 실행하지는 않습니다.

업데이트 탭도 읽기 전용입니다. `mf update --dry-run`과 같은 계획기를 사용해 `mf update --apply`를 지금 적용해도 되는지, 어떤 파일이 막고 있는지, 어떤 템플릿 파일이 생성되거나 업데이트될지 보여줍니다. 명령은 복사할 수만 있고 직접 업데이트를 적용하지는 않습니다.

실행 기록 탭도 읽기 전용입니다. mustflow가 보관하는 최신 실행 기록만 보여주며, 해당 기록 안에 남은 표준 출력과 표준 오류의 일부를 함께 표시합니다. 명령을 다시 실행하거나 원본 터미널 기록을 읽지는 않습니다.

스킬 탭도 읽기 전용입니다. `.mustflow/skills/INDEX.md`에 설치된 스킬 라우트와 연결된 `SKILL.md` 경로, 필요 입력, 수정 범위, 위험, 검증 의도, 스킬 머리말(frontmatter)과의 명령 의도 일치 여부를 보여줍니다.

문서 검수 탭은 진행 중인 검수 항목을 기본으로 보여줍니다. 승인 또는 무시된 항목은 상태 필터에서 요청했을 때만 표시됩니다.

특정 포트를 쓰려면 `--port`를 지정합니다. 서버가 시작된 뒤 기본 브라우저에서 열려면 `--open`을 사용합니다. `--no-open`은 기본 동작을 명시하고 싶은 스크립트를 위해 유지됩니다. 다른 도구가 주소를 읽어야 하면 `--json`을 사용하며, JSON 모드에서는 브라우저를 열지 않습니다.

```sh
npx mf dashboard --port 4173
npx mf dashboard --open
npx mf dashboard --no-open
npx mf dashboard --json
```

## 구조화된 출력

`--json`을 사용하면 로컬 서버가 계속 실행되는 동안 대시보드 주소, mustflow 루트, 선호 설정 파일 경로를 먼저 출력합니다.

대시보드 API는 세션별 토큰을 사용합니다. 상태 조회, 선호 설정 수정, 문서 검수 상태 전환은 모두 이 토큰이 있어야 합니다. API는 화면에 노출된 제한된 선호 설정과 문서 검수 상태 전환만 수정합니다. Git 선호값 토글은 사용자의 명시적 요청 뒤에 에이전트가 어디까지 진행해도 되는지를 나타내며, `git_commit` 같은 명령 계약 항목을 실행 가능 상태로 바꾸지는 않습니다. `git.auto_push`는 잠긴 설정으로 표시됩니다.

설정 저장에 성공하면 대시보드는 `.mustflow/config/preferences.toml`을 쓰고, `.mustflow/config/manifest.lock.toml`이 있으면 해당 파일 항목을 `last_action = "customized"`로 갱신합니다. 그래서 `mf check`, `mf status`, `mf update --dry-run`이 사용자가 승인한 로컬 선호 설정 기준선과 맞게 동작합니다.

문서 검수 작업에 성공하면 대시보드는 `.mustflow/review/docs.toml`을 갱신합니다. 검수자 종류는 `human`, `llm`, `tool`, `external`처럼 넓게만 기록하고, 검수자 ID와 요약은 자유 형식으로 남깁니다. 검수 코멘트는 검수자가 참고할 지시사항으로 표시되며 대상 문서 본문에는 섞이지 않습니다.

## 도움말 및 종료 코드

```sh
npx mf dashboard --help
```

- 종료 코드 `0`: 대시보드를 시작했거나 도움말을 출력했습니다.
- 종료 코드 `1`: 대시보드를 시작하지 못했거나 잘못된 입력이 제공되었습니다.
