mustflow documents explain the LLM-only files and fields that `mf init` creates in a user repository.

## What This Site Explains

- Where each file is placed in the target repository.
- Which files agents read first.
- What each configuration field and document section means.
- Which files are copied, which files are generated, and which files are intentionally not created.
- How command contracts prevent agents from guessing commands.
- Which context agents can inspect through `mf context --json`.

## Default Structure

```text
AGENTS.md
REPO_MAP.md  # optional generated file
.mustflow/
├─ config/
│  ├─ commands.toml
│  ├─ mustflow.toml
│  ├─ preferences.toml
│  └─ manifest.lock.toml  # generated after successful init
├─ context/
│  ├─ INDEX.md
│  └─ PROJECT.md
├─ docs/
│  └─ agent-workflow.md
├─ skills/
│  ├─ INDEX.md
│  └─ */SKILL.md
└─ state/  # generated during use
   └─ runs/latest.json
```

`mf init` does not create `README.md`, `.github/`, root `docs/`, root `skills/`, source code, or package-manager configuration.
`REPO_MAP.md` is generated from the repository structure instead of copied from the template.
`manifest.lock.toml` is generated by `mf init` to record the actual install result.
`.mustflow/state/runs/latest.json` is the latest run receipt created by `mf run`.

## Reading Order

1. Read `AGENTS.md` for the short mandatory rules.
2. Read `.mustflow/docs/agent-workflow.md` for shared work policy.
3. Read `.mustflow/config/mustflow.toml` for authoritative documents and protected paths.
4. Read `.mustflow/config/commands.toml` for executable command intents.
5. Read `.mustflow/config/preferences.toml` when present for repository-level defaults.
6. Read `.mustflow/skills/INDEX.md` to choose the relevant skill.
7. Read `.mustflow/context/INDEX.md` only when task-specific project context is needed.

This site is reference documentation. It is not copied into user projects.
