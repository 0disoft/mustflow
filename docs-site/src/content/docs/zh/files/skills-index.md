`.mustflow/skills/INDEX.md` 帮助代理在开始可重复任务前选择正确的技能文档。

## 用途

在阅读共享规则和命令契约后，如果任务匹配预定义流程，代理会参考此索引。

此文件不应包含完整流程细节。它把紧凑的路由字段连接到技能路径：触发条件、所需输入、编辑范围、风险、验证意图和预期输出。
`mf check --strict` 会将这些路由与引用的 `SKILL.md` 对比，从而发现缺失技能文档、未列出的技能、未知命令意图、命令意图漂移和路由表结构漂移。

## 选择行为

代理在任务开始和首次编辑前使用此索引。它们将用户请求和预计更改文件与列出的触发条件比较，然后在编辑该范围前阅读匹配的 `SKILL.md`。

使用技能时，或有意跳过看似适用的技能时，代理应在下一次面向用户的更新或最终报告中留下简短的选择说明。该说明属于对话报告，不应写入纳入版本管理的工作日志文件。

如果任务中出现新的条件，例如命令失败、测试契约变化或文档变化，代理应暂停并阅读新匹配的技能后再继续。

如果没有触发条件适用，代理不应臆造技能，而应继续依据 `AGENTS.md`、`.mustflow/docs/agent-workflow.md` 和 `.mustflow/config/commands.toml` 工作。

## 角色和职责

- 列出可用技能并定义精确触发条件。
- 为每条路由说明所需输入、编辑范围、风险和预期输出。
- 标明每个技能引用的命令意图。
- 保持路由紧凑，让流程细节留在各自的 `SKILL.md` 中。

## 编写准则

索引必须简洁且易于浏览。

流程细节必须放在各个 `SKILL.md` 文件中。索引只应包含帮助代理判断是否阅读某个技能以及应报告哪些证据的路由字段。

## 表结构

- **触发条件**：需要阅读该技能的任务条件。
- **技能文档**：对应 `SKILL.md` 的文件路径。
- **所需输入**：应用技能前需要的证据或请求数据。
- **编辑范围**：该技能可指导的文件或表面。
- **风险**：该路由要控制的主要失败模式。
- **验证意图**：`commands.toml` 中可能相关的意图名称。
- **预期输出**：使用技能后代理应留下的报告形态。

引入新技能时，请在此处添加路由，并确保验证意图名称与技能 frontmatter 保持同步。

## 示例：性能测量完整性

`performance-measurement-integrity-review` 是处理计数器、计时器、直方图、缓存比率、基准门禁、CPU PMU 证据、通信 IPC 结果和跨进程延迟的 adjunct route。它检查测量事件、分母与时钟域、并发快照、比较工作负载和遥测安全边界是否明确。

它补充 `performance-budget-check`：前者检查用于结论的数字是否可信，后者判断该优化是否值得和有用。两者都不会把 profiler、benchmark 或生产查询变成命令权限。
