本页不描述当前行为。它跟踪仍在评估中的提议功能。

在实现之前，这些功能不属于默认模板、命令合同或验证规则。

## 提议功能

| 项目 | 状态 | 当前决定 |
| --- | --- | --- |
| `mf dashboard` | 评估中 | 命令名已预留；实现待定。 |
| 社区 skill 仓库 | 评估中 | 外部 skills 的安装与更新规则尚未定义。 |
| Skill pack 安装 | 评估中 | 等待默认 skills 与可选 skills 的边界稳定。 |
| `.mustflow/work-items/` | 评估中 | 不包含在默认模板中；保持可选。 |
| `mf orient` | 评估中 | 当前由 `mf context`、`mf map` 和 `mf help` 覆盖。 |
| `mf refresh` | 评估中 | 当前通过 `mf update` 和 `mf check --strict` 处理指令新鲜度。 |
| 工具专用适配器 | 评估中 | 避免将工具产品名变成必需的默认文件名或强制规则。 |

## 纳入标准

功能只有在满足所有条件后，才会成为公开行为：

- 不会不必要地扩大 `mf init` 创建的默认文档流。
- 提供清晰的命令合同，使代理可预测地执行。
- 错误用法可通过 `mf check --strict` 或其他验证器检测。
- 仍然便于人类阅读和手动编辑。
