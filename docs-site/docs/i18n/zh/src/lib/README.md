# docs-site lib

语言：[英文](../../../../../src/lib/README.md) · [韩文](../../../ko/src/lib/README.md) · [中文](README.md) · [西班牙文](../../../es/src/lib/README.md) · [法文](../../../fr/src/lib/README.md) · [印地文](../../../hi/src/lib/README.md)

该目录充当文档站点的共享公共组件和工具层。它包含由多个 Astro 路由、页面或端点共享的、可重用的纯代码生成辅助工具。

---

## 架构原则：轻量级控制器 (Thin Controllers)

为了确保文档站点始终保持高性能且易于进行单元测试，本项目遵循 **轻量级控制器 / 瘦路由 (Thin Controller / Thin Route)** 架构原则：
* **无内联复杂逻辑**：路由模板和 Astro 页面代码应纯粹专注于响应处理和页面布局的组装。
* **功能解耦**：复杂的元数据处理、清单文件读取以及字符串清理任务，必须封装并隔离到此目录下的纯功能性辅助函数中。
* **可测试性**：将业务逻辑解耦为纯函数式工具，使得开发者无需启动整个 Astro 渲染流程即可独立执行轻量级单元测试。

---

## 文件注册表

* **`machine-readable.mjs`**：核心渲染引擎。它汇总来自 `../config/machine-readable.mjs` 的结构化配置，并将其翻译为适用于 `ai.txt`、`llms.txt`、`llms-full.txt` 和 `robots.txt` 端点的机器友好纯文本格式。

---

## 引入新辅助函数时的指南

添加新的实用程序或辅助代码时，请遵守以下原则：
1. 确保编写的函数是**纯函数** (对于相同的输入总是产生相同的输出，且无外部副作用)。
2. 解耦与 Astro 框架的特定关联 (例如，除非确有必要，否则避免直接引用 Astro 全局变量或浏览器特定 API)。
3. 在此注册表中更新辅助文件的功能与依赖说明。
