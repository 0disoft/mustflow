# docs-site शैलियाँ

भाषाएँ: [अंग्रेज़ी](../../../../../src/styles/README.md) · [कोरियाई](../../../ko/src/styles/README.md) · [चीनी](../../../zh/src/styles/README.md) · [स्पेनी](../../../es/src/styles/README.md) · [फ़्रांसीसी](../../../fr/src/styles/README.md) · [हिन्दी](README.md)

यह निर्देशिका दस्तावेज़ साइट के लिए दृश्य शैली परतों (visual style layers) को होस्ट करती है। अखंड यूआई फ्रेमवर्क पर भरोसा करने के बजाय, `docs-site` अपने लेआउट, घटकों और टाइपोग्राफी को विशेष लेआउट और उपयोगकर्ता संपर्क (user interaction) चिंताओं द्वारा अलग किए गए मॉड्यूल, वेनिला सीएसएस शीट का उपयोग करके प्रबंधित करता है।

---

## लोड क्रम और कैस्केडिंग (Load Order & Cascading)

इन स्टाइलशीटों का लोड और संकलन (compilation) क्रम `../config/styles.mjs` में सख्ती से परिभाषित है। नए दृश्य नियम जोड़ते समय, सुनिश्चित करें कि वे CSS कैस्केड के साथ अपेक्षित रूप से शैलियों को प्राप्त (inherit) करें।

---

## स्टाइलशीट मॉड्यूल रजिस्ट्री (Stylesheet Module Registry)

* **`tokens.css`**: डिज़ाइन टोकन इंजन। रंग पट्टियों (HSL चर), टाइपोग्राफी (Inter, Outfit फोंट), बॉर्डर त्रिज्या (border radiuses), और स्थानिक ग्रिड सिस्टम के लिए केंद्रीकृत CSS कस्टम गुणों को नियंत्रित करता है।
* **`layout.css`**: Starlight कोर फ्रेम (नेविगेशन साइडबार, सामग्री कंटेनर, हेडर) के लिए व्यूपोर्ट ग्रिड मेट्रिक्स, मार्जिन और फ्लेक्स कंटेनर को परिभाषित करता है।
* **`markdown.css`**: समृद्ध पाठ रेंडरिंग को प्रबंधित करता है। तालिकाओं, पैराग्राफ, कस्टम अलर्ट, ब्लॉककोट, इनलाइन कोड और सिंटैक्स-हाइलाइट किए गए ब्लॉक के लिए स्टाइल को नियंत्रित करता है।
* **`header-controls.css`**: विशेष रूप से नेविगेशन हेडर आइटम के लिए लेआउट को ओवरराइड करता है, जिसमें भाषा स्विचर ड्रॉपडाउन, थीम टॉगल बटन और खोज फॉर्म शामिल हैं।
* **`page-navigation.css`**: पाद लेख (footer) पृष्ठांकन लिंक के लिए नियम (जैसे, पिछला पृष्ठ / अगला पृष्ठ)।
* **`interaction.css`**: लोगो मार्क और नेविगेशन लिंक जैसे स्थिर यूआई तत्वों के लिए गतिशील सूक्ष्म-एनिमेशन, सक्रिय राज्यों, होवर प्रभावों और पाठ चयन ओवरराइड को नियंत्रित करता.
* **`accessibility.css`**: मुख्य वेब एक्सेसिबिलिटी गार्ड। उच्च-कंट्रास्ट फ़ोकस रिंग, कस्टम रूपरेखा ओवरराइड, CSS दिशा अलगाव (`unicode-bidi`), और गति को कम करने वाले मीडिया प्रश्नों (`prefers-reduced-motion: reduce`) को लागू करता है।

## डिज़ाइन टोकन सर्वोत्तम अभ्यास (`tokens.css`)

डायनामिक लाइट और डार्क थीम टॉगलिंग का सुचारू रूप से समर्थन करने के लिए हमेशा हार्डकोडेड हेक्स मानों के बजाय CSS चरों (variables) का उपयोग करें:
```css
/* अच्छा अभ्यास (Good Practice) */
.my-card {
  background-color: var(--sl-color-bg-inline);
  padding: var(--sl-spacing-md);
}

/* खराब अभ्यास (Bad Practice) */
.my-card {
  background-color: #1a1a1a;
  padding: 16px;
}
```

## पहुँच-योग्यता और कीबोर्ड नियंत्रण (Accessibility & Keyboard Controls)

स्टाइलिंग परिवर्तनों को उपयोगकर्ता सेटिंग्स का सम्मान करना चाहिए:
* **फ़ोकस स्टेट्स (Focus States)**: उच्च-दृश्यता फ़ोकस संकेतकों को बनाए रखा जाना चाहिए; `outline: none` न लिखें जब तक कि स्पष्ट रूप से समकक्ष फ़ोकस चयनकर्ता प्रदान न किया गया हो।
* **कम गति (Reduced Motion)**: जब उपयोगकर्ता के OS ने गति को अक्षम कर दिया हो, तो किसी भी कस्टम एनीमेशन, स्लाइड प्रभाव, या संक्रमण को खूबसूरती से अनुकूलित होना चाहिए। सुनिश्चित करें कि इन अपवादों को `@media (prefers-reduced-motion: reduce)` के अंदर जोड़ा गया है।
* **कीबोर्ड नेविगेशन (Keyboard Navigation)**: गैर-CSS इंटरैक्टिव राज्यों (जैसे कीबोर्ड फ़ोकस ट्रैप) को `../../public/keyboard-navigation.js` में गतिशील रूप से प्रबंधित किया जाता है और `../config/head.mjs` में पंजीकृत किया जाता है।
