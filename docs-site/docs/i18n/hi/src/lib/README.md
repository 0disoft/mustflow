यह निर्देशिका दस्तावेज़ साइट के लिए साझा उपयोगिता परत के रूप में कार्य करती है। इसमें शुद्ध, पुन: प्रयोज्य कोड जनरेशन सहायक शामिल हैं जो कई Astro रूट, पृष्ठों या एंडपॉइंट्स पर साझा किए जाते हैं।

---

## वास्तुकला सिद्धांत: पतले नियंत्रक (Architectural Principles: Thin Controllers)

दस्तावेज़ साइट को प्रदर्शनकारी और परीक्षण में आसान बनाए रखने के लिए, परियोजना **पतले नियंत्रक / पतले मार्ग (Thin Controller / Thin Route)** वास्तुकला का पालन करती है:
* **कोई इनलाइन जटिल तर्क नहीं**: रूट टेम्प्लेट और Astro पृष्ठों को पूरी तरह से अनुरोधों को हल करने और लेआउट असेंबली पर ध्यान केंद्रित करना चाहिए।
* **कार्यात्मक डिकपलिंग (Functional Decoupling)**: जटिल मेटाडेटा स्वरूपण (formatting), घोषणापत्र प्रसंस्करण (manifest processing), और स्ट्रिंग सैनिटाइजेशन इस निर्देशिका के भीतर शुद्ध सहायक कार्यों में होना चाहिए।
* **परीक्षण क्षमता (Testability)**: सहायकों को शुद्ध और पृथक रखने से पूरे Astro रेंडरिंग हार्नेस को लॉन्च किए बिना उनका यूनिट-परीक्षण किया जा सकता है।

## फ़ाइल रजिस्ट्री (File Registry)

* **`machine-readable.mjs`**: इसमें कोर रेंडरिंग engine होता है जो `../config/machine-readable.mjs` से संरचित सेटिंग्स को एकत्रित करता है और उन्हें `ai.txt`, `llms.txt`, `llms-full.txt`, और `robots.txt` एंडपॉइंट के लिए सादे टेक्स्ट प्रारूपों में अनुवादित करता है।

---

## नए सहायकों को पेश करने के दिशानिर्देश (Guidelines for Introducing New Helpers)

जब कोई नया सहायक या उपयोगिता जोड़ते हैं:
1. सुनिश्चित करें कि फ़ंक्शन **शुद्ध (pure)** है (बिना किसी दुष्प्रभाव के समान इनपुट के लिए समान आउटपुट उत्पन्न करता है)।
2. इसे Astro-विशिष्ट संदर्भों से अलग करें (उदाहरण के लिए, जब तक कि सख्त आवश्यकता न हो, Astro ग्लोबल या ब्राउज़र API को सीधे संदर्भित करने से बचें)।
3. इस रजिस्ट्री को सहायक के उद्देश्य और संदर्भ फ़ाइलों के साथ अपडेट करें।
