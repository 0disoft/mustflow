# docs-site कॉन्फ़िगरेशन

भाषाएँ: [अंग्रेज़ी](../../../../../src/config/README.md) · [कोरियाई](../../../ko/src/config/README.md) · [चीनी](../../../zh/src/config/README.md) · [स्पेनी](../../../es/src/config/README.md) · [फ़्रांसीसी](../../../fr/src/config/README.md) · [हिन्दी](README.md)

यह निर्देशिका Starlight के लिए कॉन्फ़िगरेशन घटकों को होस्ट करती है। एक एकल, अखंड कॉन्फ़िगरेशन फ़ाइल पर भरोसा करने के बजाय, `docs-site` एक मॉड्यूलराइज्ड डिज़ाइन दृष्टिकोण अपनाता है जहाँ विकल्पों को केंद्रित, डोमेन-विशिष्ट फ़ाइलों में विभाजित किया जाता है।

---

## डिज़ाइन का औचित्य (Design Rationale)

वैश्विक दस्तावेज़ साइट के रखरखाव के लिए स्थानीयकरण, स्टाइलिंग, मेटाडेटा और नेविगेशन को समवर्ती रूप से स्केल करने की आवश्यकता होती है। कॉन्फ़िगरेशन को अलग-अलग `.mjs` मॉड्यूल में विभाजित करके, हम यह सुनिश्चित करते हैं:
* **चिंताओं का अलगाव (Isolation of Concerns)**: नेविगेशन साइडबार में बदलाव से SEO मेटाडेटा या लोकेल रूट प्रभावित नहीं होते हैं.
* **सुरक्षित मर्ज (Safer Merges)**: न्यूनतम गिट संघर्ष (git conflict) जोखिमों के साथ कई योगदानकर्ता एक ही समय में अनुवाद और नेविगेशन लिंक अपडेट कर सकते हैं।

## कॉन्फ़िगरेशन फ़ाइलें मानचित्र (Configuration Files Map)

* **`site.mjs`**: मुख्य साइट मेटाडेटा शामिल करता है, जैसे कि उत्पादन URL, शीर्षक और डिफ़ॉल्ट सेटिंग्स।
* **`head.mjs`**: प्रत्येक दस्तावेज़ पृष्ठ के HTML `<head>` में इंजेक्ट किए गए कस्टम टैग को नियंत्रित करता है। विश्लेषिकी, बाहरी CDN लिंक, या साइट-व्यापी स्क्रिप्ट यहाँ पंजीकृत करें।
* **`locales.mjs`**: समर्थित अनुवाद भाषाओं (locales) और मैपिंग प्राथमिकताओं को परिभाषित करता है।
* **`machine-readable.mjs`**: `ai.txt`, `llms.txt`, `llms-full.txt`, और `robots.txt` उत्पन्न करने के लिए उपयोग किए जाने वाले सार्वजनिक मेटाडेटा मापदंडों की घोषणा करता है।
* **`navigation.mjs`**: दस्तावेज़ साइडबार संरचना, नेस्टिंग समूहों और लिंक के लिए आधिकारिक रजिस्ट्री।
* **`sidebar.mjs`**: संरचित नेविगेशन कॉन्फ़िगरेशन को सीधे Starlight में व्यवस्थित और प्रसारित करता है।
* **`styles.mjs`**: वैश्विक शैली फ़ाइलों के सख्त लोडिंग और इनहेरिटेंस क्रम को नियंत्रित करता है।
* **`starlight.mjs`**: केंद्रीय कंपोजर जो अंतिम Starlight सेटिंग्स में ऊपर दिए गए सभी मॉड्यूल को गतिशील रूप से एकत्रित करता है।

## `machine-readable.mjs` की भूमिका (AI और LLM संरेखण)

एजेंटिक एआई कोडिंग सहायकों और एलएलएम वेब क्रॉलर्स के उदय के साथ, आधुनिक दस्तावेज़ों को मशीनों द्वारा आसानी से अनुक्रमित (index) किया जाना चाहिए।
* **`llms.txt` और `llms-full.txt`**: मानकीकृत एंडपॉइंट्स जो LLM के लिए अनुकूलित संघनित कच्चे मार्कडाउन लेआउट प्रस्तुत करते हैं।
* **`ai.txt`**: विशेष रूप से डेवलपर टूल और कोडिंग एजेंटों के लिए संकेत और सख्त संदर्भ सीमाएं प्रदान करता है जो इस साइट को पढ़ रहे हैं।
* `machine-readable.mjs` में कॉन्फ़िगर किए गए गुण सीधे नियंत्रित करते हैं कि ये फ़ाइलें कैसे रेंडर की जाती हैं।

## चरण-दर-चरण रखरखाव गाइड (Step-by-step Maintenance Guides)

### साइडबार में एक नया दस्तावेज़ जोड़ना
1. अपनी नई मार्कडाउन फ़ाइल को `src/content/docs/<locale>/path/to/file.md` में लिखें या रखें।
2. `src/config/navigation.mjs` खोलें।
3. लक्ष्य अनुभाग सरणी (target section array) का पता लगाएं। अपनी प्रविष्टि ऑब्जेक्ट जोड़ें:
   ```javascript
   { label: 'My New Page', slug: 'path/to/file' }
   ```
4. सत्यापित करें कि नेविगेशन `bun run docs:check` का उपयोग करके सफलतापूर्वक निर्मित हो गया है।

### एक वैश्विक शैली फ़ाइल पंजीकृत करना
1. `src/styles/` के तहत अपनी CSS फ़ाइल बनाएं।
2. `src/config/styles.mjs` खोलें।
3. सही कैस्केड क्रम सुनिश्चित करते हुए, निर्यातित सरणी (exported array) में अपनी स्टाइलशीट का सापेक्ष पथ जोड़ें।
