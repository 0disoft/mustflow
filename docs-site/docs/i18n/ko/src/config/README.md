# docs-site 설정

언어: [영어](../../../../../src/config/README.md) · [한국어](README.md) · [중국어](../../../zh/src/config/README.md) · [스페인어](../../../es/src/config/README.md) · [프랑스어](../../../fr/src/config/README.md) · [힌디어](../../../hi/src/config/README.md)

이 디렉토리는 Starlight를 제어하는 설정 컴포넌트들을 담고 있습니다. `docs-site` 프로젝트는 하나의 거대한 설정 파일을 사용하는 대신, 역할과 영역에 따라 독립된 `.mjs` 파일로 설정을 쪼개어 관리하는 **모듈러(Modular) 설계 방식**을 채택하고 있습니다.

---

## 설계 의도

글로벌 문서 사이트를 지속적으로 유지보수하기 위해서는 다국어 로케일, 웹 스타일링, 메타데이터, 그리고 탐색(Navigation) 구조가 동시에 안전하게 확장되어야 합니다. 설정을 독립된 모듈로 조각내어 관리함으로써 얻는 이점은 다음과 같습니다:
* **관심사 분리 (Isolation of Concerns)**: 사이드바 메뉴를 수정하더라도 검색 최적화(SEO) 메타데이터나 다국어 라우팅 코드에 전혀 영향을 주지 않습니다.
* **안전한 협업**: 여러 기여자가 다국어 번역과 사이드바 링크를 동시에 수정할 때 발생할 수 있는 깃 충돌(Git Conflict) 위험을 극적으로 줄여줍니다.

---

## 설정 파일 맵 (Map)

* **`site.mjs`**: 운영 환경의 URL, 제목, 기본 구성원 설정 등 핵심 사이트 메타데이터를 관리합니다.
* **`head.mjs`**: 모든 문서 페이지의 HTML `<head>` 영역에 삽입될 커스텀 태그를 통제합니다. 외부 분석 스크립트, CDN 링크, 전역 렌더링 스크립트 등을 이곳에 등록합니다.
* **`locales.mjs`**: 지원하는 로케일 언어 목록 및 매핑 가중치를 설정합니다.
* **`machine-readable.mjs`**: `ai.txt`, `llms.txt`, `llms-full.txt`, `robots.txt`를 생성하는 데 쓰이는 공개 기계 읽기용 메타데이터를 선언합니다.
* **`navigation.mjs`**: 사이드바 메뉴의 대분류, 소분류 구조 및 문서 링크를 결정하는 중앙 레지스트리입니다.
* **`sidebar.mjs`**: 설정된 `navigation.mjs` 구조를 정제하여 Starlight 엔진에 직접 전달합니다.
* **`styles.mjs`**: 전역 CSS 파일들의 명시적 로드 순서 및 캐스케이딩(Cascading) 규칙을 관장합니다.
* **`starlight.mjs`**: 위의 모든 개별 설정 컴포넌트들을 동적으로 병합하여 최종 Starlight 환경 설정을 조율하는 중앙 지휘소 역할을 수행합니다.

## `machine-readable.mjs` 파일의 의미 (AI 및 LLM 대응)

AI 에이전트와 LLM 크롤러가 널리 보급됨에 따라, 현대의 문서 웹사이트는 기계가 쉽게 구조를 이해하고 색인할 수 있어야 합니다.
* **`llms.txt` 및 `llms-full.txt`**: LLM이 읽기에 최적화된 마크다운 구조로 사이트 요약본을 제공하는 기계용 엔드포인트입니다.
* **`ai.txt`**: 이 저장소를 탐색하는 개발용 AI 코딩 에이전트에게 특별한 기술 컨텍스트 한계 및 동작 힌트를 제공합니다.
* `machine-readable.mjs`에 선언된 설정 값들은 위 정적 파일들을 빌드 타임에 올바르게 렌더링하도록 직접 통제합니다.

## 단계별 유지보수 실무 가이드

### 사이드바에 새로운 문서 페이지 추가하기
1. 신규 마크다운 파일을 `src/content/docs/<locale>/경로/파일명.md`에 작성하여 배치합니다.
2. `src/config/navigation.mjs` 파일을 엽니다.
3. 배치할 분류 그룹 배열을 찾아 엔트리 객체를 추가합니다:
   ```javascript
   { label: '나의 새로운 페이지', slug: '경로/파일명' }
   ```
4. 저장소 루트에서 `bun run docs:check` 명령을 통해 전체 네비게이션이 에러 없이 빌드되는지 검증합니다.

### 새로운 전역 스타일시트 등록하기
1. `src/styles/` 폴더 내에 새 CSS 파일을 생성합니다.
2. `src/config/styles.mjs` 파일을 엽니다.
3. 익스포트되는 배열에 해당 스타일시트의 상대 경로를 추가하되, 스타일 상속 관계가 꼬이지 않도록 순서에 맞게 배치합니다.
