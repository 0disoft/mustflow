# docs-site lib

언어: [영어](../../../../../src/lib/README.md) · [한국어](README.md) · [중국어](../../../zh/src/lib/README.md) · [스페인어](../../../es/src/lib/README.md) · [프랑스어](../../../fr/src/lib/README.md) · [힌디어](../../../hi/src/lib/README.md)

이 디렉토리는 문서 웹사이트의 공용 유틸리티 레이어입니다. 여러 Astro 라우트, 페이지, 엔드포인트에서 범용으로 재사용되는 순수 기능성 코드 생성 헬퍼들을 보관합니다.

---

## 아키텍처 원칙: 얇은 컨트롤러 (Thin Controllers)

문서 사이트의 빌드 성능을 최상으로 유지하고 코드를 쉽게 테스트할 수 있도록, 본 프로젝트는 **얇은 컨트롤러 / 얇은 라우트 (Thin Controller / Thin Route)** 아키텍처 패러다임을 따릅니다:
* **인라인 로직 배제**: 라우트 템플릿과 Astro 페이지 코드는 순수하게 들어온 요청을 처리하고 레이아웃을 조립하는 데에만 집중해야 합니다.
* **기능적 디커플링 (Decoupling)**: 복잡한 메타데이터 가공, 매니페스트 조작, 문자열 필터링 등의 핵심 로직은 라우트 본문이 아닌 이 디렉토리 하위의 순수 헬퍼 함수 내부로 완벽히 격리해야 합니다.
* **테스트 용이성**: 비즈니스 로직을 순수 함수형 유틸로 고립시킴으로써, 전체 Astro 렌더링 서버를 띄우지 않고도 단독 유닛 테스트를 안전하고 가볍게 수행할 수 있습니다.

---

## 파일 레지스트리 (Registry)

* **`machine-readable.mjs`**: `../config/machine-readable.mjs`의 구조화된 설정을 취합하여 `ai.txt`, `llms.txt`, `llms-full.txt`, `robots.txt` 엔드포인트에 알맞은 기계 친화적 일반 텍스트 포맷으로 번역하고 렌더링하는 코어 엔진입니다.

---

## 신규 헬퍼 추가 시 가이드라인

새로운 유틸리티나 헬퍼 코드를 도입할 때는 다음을 지켜주세요:
1. 작성하는 함수는 반드시 **순수 함수(Pure Function)**여야 합니다. (동일한 입력값에 항상 동일한 결과값을 반환하며, 외부 부작용이 없을 것)
2. Astro 전역 변수나 브라우저 API에 직접 의존하지 않도록 작성하여 프레임워크 의존성을 최소화합니다.
3. 새로운 파일을 추가했다면 본 레지스트리 문서에 해당 파일의 역할과 의존 관계를 간략히 명증하여 갱신합니다.
