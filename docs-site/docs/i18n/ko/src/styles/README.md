# docs-site 스타일

언어: [영어](../../../../../src/styles/README.md) · [한국어](README.md) · [중국어](../../../zh/src/styles/README.md) · [스페인어](../../../es/src/styles/README.md) · [프랑스어](../../../fr/src/styles/README.md) · [힌디어](../../../hi/src/styles/README.md)

이 디렉토리는 문서 웹사이트의 시각적 스타일 레이어를 관리합니다. `docs-site` 프로젝트는 무거운 UI 프레임워크나 테일윈드 등의 도구에만 의존하는 대신, 독립된 바닐라 CSS 시트들을 역할과 사용자 인터랙션 영역에 맞게 조립하여 빠르고 유연한 스타일 구조를 유지합니다.

---

## 컴파일 순서 및 캐스케이딩 (Cascading)

이 스타일시트들의 명시적인 결합 및 빌드 로드 순서는 `../config/styles.mjs`에 엄격하게 정의되어 있습니다. 새로운 디자인 규칙을 추가할 때는 CSS 캐스케이딩 규칙에 따라 기존 부모 스타일이 예측 가능하게 상속되도록 설계해야 합니다.

---

## 스타일시트 모듈 레지스트리 (Registry)

* **`tokens.css`**: 디자인 시스템의 토큰 엔진입니다. 공용 컬러 팔레트(HSL 변수), 서체 스타일(Inter, Outfit 글꼴), 테두리 곡률, 반응형 레이아웃용 여백 그리드 변수 등을 선언합니다.
* **`layout.css`**: Starlight 프레임워크 코어 프레임(사이드바, 중앙 본문 영역, 상단 헤더)의 뷰포트 배치 크기와 플렉스 정렬을 통제합니다.
* **`markdown.css`**: 리치 텍스트 본문 렌더링을 관장합니다. 테이블, 문단 여백, 경고 안내창, 인용문, 코드 블록 및 문법 강조 블록 등의 디자인을 정의합니다.
* **`header-controls.css`**: 네비게이션 헤더 내부 요소(다국어 드롭다운 스위처, 다크 모드 전환 버튼, 검색 인터페이스 등)의 전용 스타일 오버라이드를 수행합니다.
* **`page-navigation.css`**: 하단 페이지 탐색(이전 페이지 / 다음 페이지로 가기) 링크의 스타일을 다룹니다.
* **`interaction.css`**: 로고 심볼이나 메뉴 링크의 마우스 오버 효과, 액티브 클릭 상태, 텍스트 선택 영역 제한 등 동적인 마이크로 애니메이션과 반응 효과를 제어합니다.
* **`accessibility.css`**: 전역 웹 접근성 안전장치입니다. 스크린 리더용 아웃라인 보존, 고대비 포커스 링, 다국어 텍스트 양방향성 격리(`unicode-bidi`), 그리고 OS 모션 효과 저감 요청 시 애니메이션을 끄는 미디어 쿼리(`prefers-reduced-motion: reduce`)를 정의합니다.

## 디자인 토큰 실무 규칙 (`tokens.css`)

동적 테마(라이트/다크 모드) 전환을 부드럽게 지원하기 위해, 색상이나 여백을 쓸 때는 정적 하드코딩 값(Hex 코드)을 배제하고 아래와 같이 반드시 선언된 CSS 전역 변수를 호출해 주세요:
```css
/* 올바른 예시 */
.my-card {
  background-color: var(--sl-color-bg-inline);
  padding: var(--sl-spacing-md);
}

/* 잘못된 예시 */
.my-card {
  background-color: #1a1a1a;
  padding: 16px;
}
```

## 접근성 및 키보드 조작 대응

새로운 디자인 요소를 얹을 때도 사용자의 브라우저 권리 설정을 엄격히 준수해야 합니다.
* **키보드 포커스(Focus)**: 포커스 상태를 명확히 보여주는 테두리 효과를 제거하지 마세요. `outline: none`을 사용해야 할 때는 동등한 성능의 가상 시각 포커스 효과를 반드시 대안으로 정의해야 합니다.
* **모션 효과 저감 (Reduced Motion)**: 사용자가 OS 설정에서 움직임 효과를 비활성화한 경우, 모든 커스텀 페이드인, 슬라이드, 트랜지션 효과도 이에 맞춰 비활성화되거나 정적으로 변하도록 디자인해야 합니다. 이 예외 처리는 반드시 `@media (prefers-reduced-motion: reduce)` 스코프 내부에 작성합니다.
* **키보드 탐색 스크립트**: CSS로 해결할 수 없는 정교한 키보드 포커스 트랩 등의 동적 행위는 `../../public/keyboard-navigation.js`에서 제어하며, `../config/head.mjs`에 등록되어 구동됩니다.
